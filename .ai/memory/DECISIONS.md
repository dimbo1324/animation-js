# Decision Log

Append-only. Never edit or delete an entry — supersede it with a new one
that references the old id. Newest last.

Format: `## D-NNN — <date> — <decision>` then Why / Consequence.

---

## D-001 — 2026-07-23 — `.ai/` is the single source of truth for all AI assistants

**Why.** Several assistants, on different models and vendors, work on this
repo in separate sessions. Duplicated instructions drift; drift produces
contradictory behaviour.

**Consequence.** `CLAUDE.md` imports the modules natively. `AGENTS.md`,
`GEMINI.md`, `.github/copilot-instructions.md`, and
`.cursor/rules/animation-js.mdc` are generated by
`scripts/ai/sync_agents.py`. Hand-editing any of them is a rule violation.
`npm run ai:check` runs in the pre-commit hook and in CI.

---

## D-002 — 2026-07-23 — Documentation and tests are opt-in

**Why.** Owner instruction. This is a craft project; effort belongs in
product logic — the engine, the scenes, the motion. Documentation and test
suites written speculatively rot faster than they help.

**Consequence.** No `README`/`*.md` authoring and no test scaffolding
without a direct request. Exception: `.ai/`, `CLAUDE.md`, generated entry
points, and `task-checklist.md`, which are infrastructure. The pre-existing
`tests/unit/helpers.test.js` was deleted in this task: it imported a module
that no longer exists, was executed by nothing, and dead code is banned.
`tests/` keeps its empty skeleton for when the owner asks.

---

## D-003 — 2026-07-23 — `new Proxy()` reactivity is the default state mechanism

**Why.** Owner instruction, and it is the right call: automatic dependency
tracking removes whole classes of over-rendering and stale-subscription
bugs that manual observers reintroduce every time.

**Consequence.** `src/core/reactive.js` implements `reactive`, `effect`,
and `computed` with an equality guard, frame-batched flushing, a `WeakMap`
proxy cache, and lazy deep wrapping. Manual `render()` broadcasts are
forbidden.

**Boundary.** Reactivity drives structure and configuration. Per-entity
simulation state stays in plain objects mutated in `onUpdate` — proxy traps
on a thousand entities at 60 fps would cost exactly what this exists to
save. Restated in `.ai/project/13-reactivity-doctrine.md`.

---

## D-004 — 2026-07-23 — Shell and scenes are separated by a mount element

**Why.** The owner wants background/chrome and animation code developed
independently, without the two mixing.

**Consequence.** `src/shell/` exposes a `stage` element and never looks
inside it. `src/scenes/*` own everything under their root and never look
outside it. `src/core/SceneHost.js` is the only bridge. Neither side may
import the other; the guardrails job in CI greps for violations. Adding an
animation touches two registration lines and no shell code.

---

## D-005 — 2026-07-23 — One persistent rAF loop; batching uses one-shot frames

**Why.** Parallel loops desynchronise, duplicate layout work, and make a
frame budget impossible to reason about.

**Consequence.** `src/core/Ticker.js` owns the only persistent loop. Scenes
subscribe; opening a private `requestAnimationFrame` loop inside
`src/scenes/` is an ESLint error. `src/core/scheduler.js` may schedule a
one-shot frame to drain its queues, and the ticker drains them itself while
running so no second callback is paid for.

---

## D-006 — 2026-07-23 — Local git hooks, not husky

**Why.** Husky adds a dependency and an indirection layer for something git
does natively. `core.hooksPath` is one config line and the hooks stay
tracked, readable, and dependency-free.

**Consequence.** `.githooks/` holds `reference-transaction` (refuses
deletion of local `main`), `pre-commit`, `commit-msg`, and `pre-push`.
`npm install` wires them up through `scripts/setup/install-git-hooks.mjs`.
The husky dependency and `.husky/` were removed.

---

## D-007 — 2026-07-23 — Formatting is vertical-first and deliberately strict

**Why.** Owner instruction: code should read top-to-bottom, one idea per
line.

**Consequence.** `printWidth: 72`, `singleAttributePerLine: true`,
`trailingComma: "all"`, `experimentalOperatorPosition: "start"`. ESLint
runs with `--max-warnings=0`, so every rule is effectively an error.
Stylelint enforces BEM class names, no `!important` without an inline
justification, no ID selectors, and canonical property order. Relaxing any
of these to make a check pass is forbidden; fix the code instead.

---

## D-008 — 2026-07-23 — `package-lock.json` is tracked

**Why.** CI runs `npm ci`, which requires it, and tooling installs should
be reproducible across machines.

**Consequence.** Removed from `.gitignore`. `.vscode/` is tracked for the
same reason: shared formatter and linter settings.
