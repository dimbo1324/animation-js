# Codex Project Automation

Project-scoped Codex configuration for animation-js.

Durable rules live in the shared modules under `.ai/`. Codex reads them
through `AGENTS.md`, which is GENERATED by concatenating those modules —
never edit `AGENTS.md` by hand. Edit a module, run `npm run ai:sync`, and
commit both together. Claude Code imports the same modules natively via
`CLAUDE.md`, so both assistants always follow identical rules.

## Files

- `config.toml` — project defaults for model reasoning and agent limits.
- `agents/` — custom agents for focused delegation; name-for-name mirror of
  `.claude/agents/`.
- `skills/` — reusable project workflows; mirror of `.claude/skills/`.

## Delegation map

Use one main thread as the task owner and spawn agents only for independent
work.

| Agent                  | Use for                                              |
| ---------------------- | ---------------------------------------------------- |
| `anim-engine`          | `src/core/` — ticker, reactive, DOM batching, Scene  |
| `anim-scene-builder`   | `src/scenes/` — characters, rigs, motion             |
| `anim-shell-ux`        | `src/shell/` + `src/styles/` — chrome, theme, UX     |
| `anim-perf-auditor`    | Frame-budget review of a scene or a hot path         |
| `anim-code-reviewer`   | Reviewing a diff in a fresh context before finishing |
| `anim-repo-maintainer` | Tooling, CI, hooks, rule-module upkeep               |

## Quality shortcuts
